An Ontology-driven Service-oriented System for ESG Metric Computation and Reporting

422K+ ESG Records

This project presents an ontology-driven ESG metric system built on the ESG Metric Knowledge Graph (ESGMKG). The system operationalises ESG metric knowledge to support context-aware metric discovery, dependency-aware computation, and provenance-preserving ESG report generation. Rather than relying on hard-coded logic, the system interprets ontology-defined relationships at runtime to identify applicable metrics, distinguish between directly measured and derived metrics, resolve metric dependencies, and coordinate access to datasets and executable calculation models. The current prototype is evaluated using ESG and financial data for 48 semiconductor companies. For the GHG Emission Intensity example reported in the paper, the corresponding ESGMKG execution path is represented by 14 RDF triples linking the industry, framework, category, metric, calculation model, dataset variables, data sources, and executable implementation.

Service-Oriented Architecture

Knowledge Graph Service

Provides structured access to ESGMKG through RDF and SPARQL queries. It retrieves ontology-defined relationships among industries, reporting frameworks, categories, metrics, dependencies, and calculation models.

Data Retrieval Service

Retrieves ESG and financial data values from external datasets based on mappings defined in ESGMKG. It supports consistent access to company- and year-specific inputs required for metric computation.

Calculation Service

Performs dependency-aware metric computation. It determines whether a metric is directly measured or derived, resolves required inputs recursively, and invokes the corresponding executable model implementation.

Report Service

Generates structured ESG reports by combining computed metric values with contextual metadata and provenance information, including data lineage and applied calculation methods.
